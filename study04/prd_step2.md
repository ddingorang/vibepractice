# PRD Step 2 — 레시피 생성

## 목표
Step 1에서 추출한 식재료 목록을 바탕으로 `google/gemma-3-27b-it:free` 모델을 사용해 실용적인 레시피를 생성하고 사용자에게 제안한다.

## 사용자 스토리
- 사용자는 인식된 재료를 기반으로 만들 수 있는 요리 목록을 추천받는다.
- 사용자는 원하는 레시피를 선택해 상세 조리법을 확인한다.
- 사용자는 레시피를 저장하거나 다른 레시피를 다시 추천받을 수 있다.

## 기능 요구사항

### 레시피 추천 요청
- Step 1의 재료 목록과 동일한 모델(`google/gemma-3-27b-it:free`)을 사용
- 프롬프트: 재료를 넘기고 JSON 형식으로 레시피 3개를 반환하도록 지시
  ```
  다음 재료로 만들 수 있는 요리를 3가지 추천해줘: {재료 목록}
  반드시 아래 JSON 형식으로만 응답해:
  {
    "recipes": [
      {
        "name": "요리명",
        "time": "조리 시간(분)",
        "difficulty": "쉬움|보통|어려움",
        "ingredients": ["재료1(양)", ...],
        "steps": ["1. ...", "2. ...", ...]
      }
    ]
  }
  ```

### 레시피 목록 표시
- 추천된 레시피 3개를 카드 형태로 나열
- 각 카드에 요리명, 조리 시간, 난이도 표시
- 카드 클릭 시 상세 조리법 모달 또는 상세 페이지로 이동

### 레시피 상세 화면
- 재료 목록 (필요량 포함)
- 단계별 조리법 (번호 목록)
- "저장하기" 버튼 (Step 3 연동)
- "다시 추천받기" 버튼 (동일 재료로 재요청)

## API 명세

### POST /api/recipes
- Request:
  ```json
  { "ingredients": ["달걀", "우유", "당근", "양파"] }
  ```
- Response:
  ```json
  {
    "recipes": [
      {
        "name": "달걀볶음밥",
        "time": "15",
        "difficulty": "쉬움",
        "ingredients": ["달걀 2개", "밥 1공기", "당근 1/4개"],
        "steps": ["1. 당근을 잘게 썬다.", "2. ..."]
      }
    ]
  }
  ```

## 비기능 요구사항
- Step 1 → Step 2 전환 시 재료 목록을 URL 파라미터 또는 세션 스토리지로 전달
- 레시피 생성 응답 시간 목표: 15초 이내
- JSON 파싱 실패 시 원문 텍스트로 fallback 표시
- rate limit 발생 시 자동 재시도 (최대 3회, 5초 간격)
