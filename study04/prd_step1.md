# PRD Step 1 — 냉장고 이미지 인식

## 목표
사용자가 냉장고 사진을 업로드하면 `google/gemma-3-27b-it:free` 모델을 통해 식재료 목록을 추출한다.

## 사용자 스토리
- 사용자는 냉장고 사진을 업로드하거나 카메라로 촬영한 이미지를 제출할 수 있다.
- 사용자는 AI가 인식한 식재료 목록을 확인하고, 잘못 인식된 항목을 수정하거나 누락된 항목을 추가할 수 있다.

## 기능 요구사항

### 이미지 업로드
- JPG, PNG, WEBP 형식 지원, 최대 10MB
- 드래그 앤 드롭 및 파일 선택 버튼 제공
- 업로드 전 미리보기 표시

### AI 이미지 분석
- 모델: `google/gemma-3-27b-it:free` (OpenRouter API)
- 프롬프트: 이미지 내 식재료를 JSON 배열로 반환하도록 지시
  ```
  이 냉장고 이미지에서 식재료를 모두 찾아줘.
  반드시 아래 JSON 형식으로만 응답해:
  {"ingredients": ["재료1", "재료2", ...]}
  ```
- 응답에서 JSON 파싱 후 재료 목록으로 변환

### 결과 표시 및 편집
- 인식된 재료를 태그(chip) 형태로 표시
- 각 태그 옆 삭제(×) 버튼으로 제거 가능
- 텍스트 입력으로 재료 직접 추가 가능
- "레시피 추천받기" 버튼으로 Step 2로 진행

## API 명세

### POST /api/analyze
- Request: `multipart/form-data` — `image` 파일
- Response:
  ```json
  {
    "ingredients": ["달걀", "우유", "당근", "양파"]
  }
  ```
- 에러 응답:
  ```json
  { "error": "이미지를 인식할 수 없습니다." }
  ```

## 비기능 요구사항
- 분석 응답 시간 목표: 10초 이내
- rate limit 발생 시 자동 재시도 (최대 3회, 5초 간격)
- 분석 중 로딩 상태 표시로 사용자 이탈 방지
